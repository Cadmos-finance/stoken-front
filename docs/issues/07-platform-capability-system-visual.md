## What to build

Redesign the existing "What we do" sphere concept into a clearer Stoken platform-system visual. The completed slice should show Origination, Rating, Tokenization, and Distribution as four coordinated platform capabilities, connected through a controlled central Stoken platform node.

## Acceptance criteria

- [ ] Start with one RED browser-visible behavior test that proves all four capabilities render with clear labels and concise descriptions.
- [ ] The visual retains the "four capabilities, one platform" concept while feeling like a financial operating system rather than a decorative orb.
- [ ] Origination, Rating, Tokenization, and Distribution are visually connected to the Stoken platform node.
- [ ] Motion, lines, nodes, and data accents are subtle and controlled.
- [ ] The section works as a standalone homepage section and remains readable on mobile.
- [ ] Copy is tightened but preserves the approved meaning of each capability.
- [ ] Refactor only after the browser-visible behavior test is green; tests assert visible capability behavior and content, not drawing implementation details.
